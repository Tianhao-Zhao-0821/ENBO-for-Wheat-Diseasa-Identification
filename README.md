
# ENBO: A Novel Deep Learning Classification Model for Wheat Disease Identification

> **论文状态**：处于投刊阶段  
> **论文标题**：_ENBO: A Novel Deep Learning Classification Model for Wheat Disease Identification_  
> **核心思想**：在 EfficientNetV2 基础上，通过引入 **BAM 双注意力模块** 与 **ODConv 全维动态卷积**，显著提升小麦叶片病害识别的精度与鲁棒性，为智慧农业提供高效、轻量化的病害诊断解决方案。（官方 PyTorch 实现）

----------
## 1. 研究背景与模型定位

小麦作为全球核心粮食作物，其叶片病害（如白粉病、叶枯病、黄斑叶枯病、叶锈病）严重威胁粮食安全与农业经济稳定。传统人工检测存在动态监测能力不足、早期预警滞后、诊断精度受限等问题。

本文提出一种基于改进 EfficientNetV2 的病害识别模型 **ENBO**。该模型以 EfficientNetV2‑S 为基础网络，通过优化深度与宽度配置，引入 **BAM 双注意力模块** 与 **ODConv 全维动态卷积**，解决小麦叶片病害“病灶形态多变、局部‑全局特征融合效率低、相似病害区分精度不足”的核心问题。最终在自建小麦叶片病害数据集（WPLDD）上实现 **99.4%** 的分类准确率，参数量仅 **10.57M**，推理速度达 **2.42 ms/帧**（约 413 fps），为小麦病害田间自动化诊断提供轻量化、高精度、强鲁棒性的技术方案。

----------

## 2. ENBO 核心创新点

### 1.双注意力机制融合

-   **BAM 注意力模块**：通过并行通道注意力与空间注意力分支，生成三维注意力图，强化模型对病害关键区域的定位能力。通道分支利用全局平均池化与MLP捕捉通道间依赖，空间分支通过空洞卷积扩大感受野，精准聚焦白粉病霉层、叶锈病孢子堆等不规则病灶区域，有效抑制复杂背景干扰。
    
-   **ODConv 全维动态卷积**：在卷积核的四个维度（空间位置、输入通道、输出通道、卷积核个数）引入动态注意力机制，实现卷积核参数的自适应调整。通过为每个输入样本动态生成专属卷积核，增强模型对多尺度病灶形态（如早期微小病斑与后期大面积发病）的建模能力，提升对相似病害（如黄斑叶枯病与叶锈病）的区分精度。
    

### 2.分层渐进式特征融合

通过在不同层级策略性嵌入 BAM 与 ODConv 模块，构建多粒度特征表示。浅层（Stage 1‑2）采用 Fused‑MBConv 保持高效计算，聚焦纹理与颜色特征；中层（Stage 3）引入 ODConv 增强多尺度感知能力；深层（Stage 4‑6）同时集成 ODConv 与 BAM，强化病灶形态与空间模式识别。跨阶段连接机制确保梯度流畅传播与特征复用，实现从局部细节到全局语义的渐进式特征融合，适配小麦叶片病害的多尺度检测需求。

### 3.效率与精度平衡

通过优化网络深度与宽度配置、引入轻量化注意力模块及动态卷积，模型参数量降至 **10.57M**，计算量为 **6.81G FLOPs**，推理速度达 **2.42 ms/帧**（约 413 fps）。在自建数据集上实现 **99.4%** 的识别准确率，较基线 EfficientNetV2‑S 提升 3.7 个百分点，参数量降低 50.8%。轻量化设计可高效部署于 Jetson Nano、树莓派等农业边缘设备，满足便携式诊断终端、无人机载检测模块、地面机器人视觉系统的实时性要求。
## 3. 实验数据集：WPLDD

### 3.1 数据集概况

本研究基于自建小麦叶片常见病害检测数据集（Wheat Leaf Disease Detection Dataset, WPLDD），数据集已随项目上传至仓库的 WPLDD/ 文件夹，无需额外下载。

|数据集名称|包含类别|图像总数|图像分辨率|数据分布（训练:验证:测试）|
|-|-|-|-|-|
|WPLDD|白粉病、叶枯病、黄斑叶枯病、叶锈病、健康叶片|7394 张|统一 resize 至 384×384|3:1:1（通过代码自动划分）|

### 3.2 数据集结构

仓库中`` WPLDD/ ``文件夹组织如下，图像均采集于小麦主产区田间（涵盖不同生育期、光照与拍摄角度），可直接用于模型训练/测试：

`WPLDD/`
`├── 白粉病/       # 小麦白粉病叶片图像（病斑特征：叶片表面覆盖白色粉状霉层，阻碍气体交换与光合作用，导致叶片逐渐变黄、枯萎）`
`├── 叶枯病/       # 小麦叶枯病叶片图像（病斑特征：叶片枯萎、变色，严重时大面积干枯，影响植株生理功能）`
`├── 叶锈病/       # 小麦叶锈病叶片图像（病斑特征：产生锈色孢子堆，初期黄色或橙色，后期深褐色，使叶片失去光泽、功能衰退）`
`├── 黄斑叶枯病/   # 小麦黄斑叶枯病叶片图像（病斑特征：初期为褐色小斑点，随病情发展扩大并连接成片，对叶片损害较大）`
`└── 健康植株/     # 健康小麦植株图像（特征：叶片鲜绿、形态舒展有光泽，整体生长旺盛，无病害或虫害迹象）`

-   **CPU**：支持推理测试（单张图像推理耗时约 0.5‑1 秒），但训练耗时显著增加（约 20‑25 小时），不推荐用于完整训练流程。
## 5. 实验结果

### 5.1 核心指标对比（WPLDD 数据集）

ENBO 与主流深度学习模型在小麦叶片病害分类任务上的性能对比如下，模型在精度、参数量、计算效率上实现最优平衡，尤其对相似病害（黄斑叶枯病/叶锈病、健康叶片/早期叶枯病）的区分能力显著提升：

|模型(Model)|分类准确率(Accuracy)|计算量(FLOPs)|参数量(M)|
|-|-|-|-|
|VGG16|84.91%|15.5G|138.4|
|ZFNet|87.35%|11.3G|62.4|
|ResNet50|89.24%|4.1G|25.6|
|MobileNetV3|92.10%|0.22G|5.4|
|Swin Transformer（Tiny）|96.42%|4.5G|28.3|
|**ENBO（本文）**|**99.4%**|**6.81G**|**10.57**|

**注：**

-   准确率为多次重复实验的平均值，结果稳定性高，最高达 99.4%；
    
-   相较于 Swin Transformer（Tiny），ENBO 参数量减少 **62.6%**（从 28.3M 降至 10.57M），准确率提升 **2.98 个百分点**；
    
-   在相似病害区分上，基线模型（如 ResNet50、VGG16）对黄斑叶枯病与叶锈病、健康叶片与早期叶枯病的混淆率较高（混淆率 >5%），而 ENBO 通过 BAM 与 ODConv 双注意力协同机制，将上述类别的混淆率降至 **2% 以下**，显著提升区分精度；
    
-   模型推理速度达 **2.42 ms/帧**（约 413 fps），在保持高精度的同时实现轻量化设计，可高效部署于农业边缘设备。
## 6. 代码使用说明

### 6.1 模型训练

运行 `train.py` 脚本启动训练，支持通过参数调整训练配置，示例命令（适配小麦叶片病害数据集）：

`python train.py \`
 `--data-path ./WPLDD \`
 `--num-classes 5 \`
` --epochs 100 \`
` --batch-size 8 \`
` --lr 0.001 \`
 `--use-bam True \`
 `--use-odconv True \`
 `--device cuda:0`

**关键参数说明：**

|参数名|含义|默认值|
|-|-|-|
|`--data-path`|数据集根目录路径（需包含训练图像及标签）|`./WPLDD`
|`--num-classes`|分类类别数|`5`
|`--epochs`|训练轮数|`100`
|`--batch-size`|批次大小（根据 GPU 显存调整，推荐 8/16）|`8`
|`--lr`|初始学习率|`0.001`
|`--lrf`|最终学习率比例（余弦退火参数）|`0.1`
|`--use-bam`|是否启用 BAM 注意力模块|`True`|
|`--use-odconv`|是否启用 ODConv 动态卷积模块|`True`
|`--weights`|预训练权重路径（可选）|`''`
|`--freeze-layers`|是否冻结骨干网络（仅训练分类头）|`False`
|`--device`|训练设备（`cuda:0` 或 `cpu`）|`cuda:0`

**训练输出：**

-   训练过程中，模型自动保存验证集准确率最高的权重至 `./weights/` 目录，文件名为 `best_model.pth`；
    
-   每轮训练结束后保存当前权重，文件名为 `model-{epoch}.pth`；
    
-   训练日志（损失值、准确率、学习率）实时打印；
    
-   TensorBoard 日志保存至 `runs/` 目录，可通过 `tensorboard --logdir=runs` 可视化训练过程。
    

### 6.2 模型预测

#### 单张图像预测

使用训练好的权重进行单张小麦叶片图像预测，运行 `predict.py` 脚本，示例命令：

`python predict.py \`
` --image-path ./test.jpg \`
` --weights ./weights/best_model.pth \`
 `--device cuda:0`

预测输出示例：

`输入图像：./test.jpg`
`预测类别：小麦白粉病`
`置信度：0.9987`

#### 批量预测与评估

如需对整个文件夹进行预测并统计准确率，可修改 `predict.py` 中的 `main_folder_path` 变量为目标文件夹路径，运行后将自动遍历所有子文件夹，输出各文件夹准确率、总体准确率及混淆矩阵，并保存详细结果到 `folder_prediction_results.txt`。

### 6.3 错误样本分析

使用 `select_incorrect_samples.py` 筛选验证集中预测错误的样本，用于模型诊断与数据清洗，示例命令：

`python select_incorrect_samples.py \`
` --data-path ./WPLDD \`
` --weights ./weights/best_model.pth \`
` --batch-size 8 \`
` --device cuda:0`

**输出文件：**

-   `error_records.csv`：UTF-8 编码，包含 `file_path`, `true_label`, `predicted_label` 三列；
    
-   控制台输出各类别准确率及总体准确率统计，帮助定位易混淆类别。

## 8. 已知问题与注意事项

问题类型

说明与解决方案

数据集适配

当前模型与权重仅针对上述 5 类小麦病害（白粉病、叶枯病、叶锈病、黄斑叶枯病、健康）训练。若需新增类别，请补充对应类别图像（建议每类 ≥500 张）并修改 `train.py` 中的 `--num-classes` 参数后重新训练。

图像分辨率

输入图像在预处理阶段自动 resize 至 384×384。若原始图像分辨率过低（<256×256），早期微小病斑特征可能丢失，建议采集分辨率 ≥512×512 的原始图像以保证识别效果。

注意力模块依赖

`bam.py` 与 `odconv.py` 为自定义模块，若缺失则模型无法正常运行。完整功能需确保两文件与 `model.py` 存在于同级目录。代码已内置模块存在性检查，缺失时会报错提示。

CUDA 兼容性

默认使用 `cuda:0` 进行训练与推理。若出现 CUDA 不兼容问题，可将 `--device` 参数改为 `cpu`，但训练耗时将显著增加（约 20‑25 倍），推理速度降至约 0.5‑1 秒/张。

显存优化

训练时若显存不足（如 GPU 显存 <6GB），可采取以下措施：  
• 降低 `--batch-size` 至 4 或 2；  
• 启用 `--freeze-layers` 冻结骨干网络，仅训练分类头；  
• 使用梯度累积（需手动修改训练脚本）。

田间场景适配

实际田间检测时，建议通过数据增强策略（随机裁剪、水平翻转、颜色抖动、亮度调整等）扩充训练集，提升模型对光照变化、拍摄角度、复杂背景的鲁棒性。可在 `my_dataset.py` 中自定义数据增强流程。

类别混淆

实验中发现健康叶片与早期叶枯病、黄斑叶枯病与叶锈病存在一定视觉相似性。若混淆率较高，建议使用 `select_incorrect_samples.py` 分析错误模式，针对性优化对应类别的数据质量（如增加样本多样性、清洗标注错误样本）。

预训练权重

若使用预训练权重 `pre_efficientnetv2-s.pth`，请确保其与模型结构匹配。该权重为 ImageNet 预训练的 EfficientNetV2‑S 权重，可用于迁移学习加速收敛。若从头训练，可忽略该参数。
